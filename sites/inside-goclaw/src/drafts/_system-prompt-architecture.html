01 — Vấn đề

Prompt không phải string

Khi bạn build một AI agent production, system prompt không đơn giản là một string dài. Nó phải đáp ứng nhiều yêu cầu xung đột nhau — về nội dung, performance, và maintainability.

Về nội dung

Main agent cần full instructions. Subagent chỉ cần subset nhỏ. Heartbeat monitor cần prompt cực kỳ slim. Cron job cần instructions khác hoàn toàn.

Về performance

Token nhiều = latency cao. Prompt thay đổi mỗi request → không cache được. Anthropic caching tiết kiệm 90% chi phí — nhưng phải biết phần nào stable.

Về maintainability

Prompt logic rải rác trong codebase là nightmare. Provider A cần reasoning format, Provider B cần plain text. Predefined vs open agent cần framing khác nhau.

Naive approach

Một string template khổng lồ với hàng trăm if/else. Không scale. Không test được. Không cache được.

Giải pháp: GoClaw dùng composable prompt blocks — kiến trúc lắp ghép modular. Mỗi section là một Go function độc lập. BuildSystemPrompt() ghép tất cả lại theo thứ tự cố định.

05 — Cache Boundary

Chia stable vs dynamic

Anthropic prompt caching hoạt động theo hash. Nếu prompt thay đổi mỗi request → không cache → pay full price. Trong GoClaw, 80-90% system prompt là stable. Giải pháp: một marker chia prompt thành 2 block.

90%
Chi phí tiết kiệm

~20

↑ Stable — cache_control: ephemeral

↓ Dynamic — no cache_control
buildTimeSection() extraContext USER.md BOOTSTRAP.md DELEGATION.md TEAM.md buildPersonaReminder()

Determinism là bắt buộc

Để cache hit, output phải hoàn toàn deterministic với cùng input. Tool names được sort trước khi inject. Time section dùng date-only.

06 — Provider Contributions

Customize mà không sửa core

Khác nhau giữa các LLM provider đôi khi cần inject logic riêng vào prompt. GoClaw giải quyết bằng PromptContributor interface — provider implement, core không thay đổi.

StablePrefix
string
Inject trước cache boundary → là một phần của stable block → cache được
DynamicSuffix
string
Inject sau cache boundary → per-turn context: user locale, conversation metadata
SectionOverrides
map[string]string
Override section theo ID. Chỉ 2 sections: execution_bias, tool_call_style. Safety bị blocked.
internal/providers/prompt_contribution.go
type PromptContribution struct { StablePrefix string // trước cache boundary DynamicSuffix string // sau cache boundary SectionOverrides map[string]string // override section bằng ID } type PromptContributor interface { PromptContribution() *PromptContribution } // GPT-5.4 cần <think>...</think> tags contribution := &PromptContribution{ StablePrefix: "## Reasoning Format\nUse <think>...</think> before each response.", }
Safety section bị blocked: Không provider nào được override safety rules. Đây là hard constraint trong code, không phải convention.

08 — Context-Aware Assembly

Mỗi agent khác nhau

SystemPromptConfig có 50+ fields. Assembly thay đổi theo context agent — không chỉ theo mode.

Bootstrap Detection

Khi BOOTSTRAP.md có trong context files → first-run mode. Chỉ có write_file, không có skills/MCP/team/spawn. Sau onboarding, BOOTSTRAP.md bị cleanup — lần sau không còn bootstrap mode.

Predefined vs Open Agent

Predefined agents wrap context files với <internal_config> tag. Open agents dùng <context_file> tag. Framing khác nhau → model treat content khác nhau → confidentiality hoạt động tốt hơn.

Group Chat Detection

PeerKind == "group" inject hint: không reply khi người dùng đang nói với người khác. ChatTitle được sanitize (strip quotes, newlines, truncate 100 chars) để prevent prompt injection.

Orchestration Mode

ModeSpawn: hide delegate + team_tasks. ModeDelegate: Delegation Targets section. ModeTeam: Team Workspace + Team Members sections. Mỗi mode unlock thêm tools và sections.

09 — PromptBuilder Interface

Hướng tới template engine

PromptBuilder interface được thiết kế cho future extensibility. BridgePromptBuilder hiện tại map PromptConfig → SystemPromptConfig → BuildSystemPrompt() — bridge pattern giữ backward compat.

internal/agent/prompt_config_types.go — v3 interface
type PromptConfig struct { Identity bool Persona bool Instructions bool Tools bool Skills bool Team bool Workspace bool Memory bool Sandbox bool Orchestration bool Mode PromptMode ProviderVariant string // "" = default, "codex", "dashscope" // Data payloads per section IdentityData IdentityData PersonaContent string ToolsData ToolsSectionData // ... }
Bridge Pattern: Khi cần A/B testing hoặc provider-specific templates, swap BridgePromptBuilder bằng TemplatePromptBuilder mà không thay đổi gì ở pipeline layer.
