01 — Vấn đề

Agent truyền thống không thể cải thiện

Hầu hết AI agent hoạt động theo vòng lặp cố định: nhận input → suy nghĩ → trả lời. Khi workflow phức tạp lặp lại, agent vẫn phải "học lại từ đầu" mỗi lượt — không nhớ rằng mình đã làm điều này hàng chục lần trước.

Không có cơ chế nào để: nhớ pattern (workflow nào hiệu quả, tool nào hay fail), phản ánh (nhận ra sự lặp lại và đóng gói), thích nghi (điều chỉnh hành vi dựa trên feedback thực tế). GoClaw giải quyết từng bài toán bằng một cơ chế riêng biệt.

02 — Tầng 1: Self-Evolution

SOUL.md — Tiến hóa phong cách giao tiếp

Mỗi predefined agent có file SOUL.md định nghĩa tone, giọng văn, cách phản hồi. Khi self_evolve = true, agent được phép tự cập nhật file này — dùng chính write_file tool vốn có, được intercepted bởi ContextFileInterceptor để route vào PostgreSQL.

Giới hạn an toàn — CAN / MUST NOT

internal/agent/systemprompt.go — buildSelfEvolveSection()
You may update SOUL.md to refine communication style (tone, voice, vocabulary, response style). What you CAN evolve: - Tone, voice, and manner of speaking - Response style and formatting preferences - Vocabulary and phrasing patterns - Interaction patterns based on user feedback What you MUST NOT change: - Your name, identity, or contact information - Your core purpose or role - Any content in IDENTITY.md or AGENTS.md (these remain locked)

Enforcement ở 3 tầng

1
System Prompt
systemprompt.go — CAN/MUST NOT guidance inject khi mỗi request
2
Context File Interceptor
context_file_interceptor.go — Chỉ SOUL.md và CAPABILITIES.md mới writable
3
File Locking
IDENTITY.md, AGENTS.md luôn read-only — không thể sửa qua bất kỳ path nào
Lưu ý: Agent chỉ có thể thay đổi cách nó nói chuyện, không thể thay đổi nó là ai. Chi phí token: ~95 tokens/request. Chỉ áp dụng với predefined agents.

03 — Tầng 2: Skill Learning Loop

Đóng gói workflow thành Skill

Khi một predefined agent hoàn thành task phức tạp (nhiều tool calls), hệ thống không để workflow đó biến mất. Ba điểm chạm trong agent loop để phát hiện và đóng gói pattern.

Skill Learning Flow
Admin bật skill_evolve = true
↓
System prompt: hướng dẫn SHOULD/SHOULD NOT tạo skill
↓
Agent xử lý task (think → act → observe)
↓
Budget milestone nudge (70% / 90%)
↓
Task hoàn thành — totalToolCalls ≥ skill_nudge_interval?
↓
Không
Response bình thường
Có
Postscript: "Save as skill?"
↓
User "save as skill" → skill_manage(create) → Auto-granted

Postscript — Lời đề nghị cuối bài

internal/agent/loop_pipeline_callbacks.go
--- _This task involved several steps. Want me to save the process as a reusable skill? Reply "save as skill" or "skip"._
Nguyên tắc: Không có skill nào được tạo mà không có sự đồng ý tường minh của user. Postscript chỉ là lời đề nghị — user phải reply "save as skill" thì agent mới gọi skill_manage(create).

skill_manage Tool — Actions

Action Params Mô tả
createcontent (SKILL.md)Tạo skill mới từ workflow
patchslug, find, replaceCập nhật một phần skill đã tạo
deleteslugXóa mềm, move vào .trash/
Tool Gating

Khi skill_evolve = false, skill_manage hoàn toàn ẩn với LLM — bị filter khỏi toolDefs trước khi gửi lên provider và khỏi toolNames trong system prompt. Tool vẫn tồn tại trong shared registry (admin thấy), nhưng agent không có awareness về nó.

04 — Budget Nudges

Ephemeral messages giữa agent loop

Khi ngưỡng iteration bị đạt, hai ephemeral nudge được inject vào giữa agent loop — không được persist vào session history, không tốn token về sau. Vai trò: nhắc agent suy nghĩ về khả năng tái sử dụng trong khi vẫn còn budget.

[System] You are at 70% of your iteration budget. Consider whether any patterns from this session would make a good skill.
[System] You are at 90% of your iteration budget. If this session involved reusable patterns, consider saving them as a skill before completing.
Task hoàn thành → Postscript khi totalToolCalls ≥ skill_nudge_interval (default: 15)

Versioning Skills — Immutable per version

skills-store/ — filesystem layout
skills-store/ ├── my-workflow-skill/ │ ├── 1/ │ │ └── SKILL.md │ └── 2/ ← patch tạo version mới │ ├── SKILL.md │ └── scripts/ (copy từ v1) └── .trash/ └── old-skill.1710000000 ← soft-deleted
Concurrency Control

pg_advisory_xact_lock keyed on FNV-64a hash của slug — serialize concurrent version creation cho cùng một skill. Database dùng ON CONFLICT(slug) DO UPDATE + RETURNING id để handle upserts atomically.

05 — Tầng 3: Metrics & Suggestions

Cải thiện dựa trên dữ liệu 7-day window

Trong quá trình agent hoạt động, GoClaw ghi lại ba loại metrics. SuggestionEngine chạy như cron job hàng ngày, phân tích và tạo suggestions cho admin review.

Loại Dữ liệu thu thập Ví dụ
toolinvocation_count, success_rate, failure_count, avg_duration_msweb_search gọi 150 lần, 85% thành công
retrievalrecall_rate, precision, relevance_score, query_countKnowledge vault: 200 queries, 18% usage rate
feedbackrating, sentiment, effectiveness_scoreUser abort/rephrase pattern

Ba quy tắc phân tích — Pluggable Rules

📉
LowRetrievalUsageRule
Trigger: usage_rate < 0.2 trên 50+ queries — knowledge vault trả kết quả nhưng agent dùng dưới 20%.
→ Đề xuất tăng retrieval threshold để lọc ketat hơn
💥
ToolFailureRule
Trigger: success_rate < 0.1 trên 20+ calls — tool fail hơn 90% lần gọi.
→ Đề xuất review config hoặc disable tool
🔄
RepeatedToolRule
Trigger: tool gọi > 100 lần/tuần với success rate > 50%.
→ Đề xuất tạo skill + tự generate skill_draft template
Duplicate Prevention

Mỗi suggestion được identify bằng composite key (type, metric_key). Nếu đã có pending suggestion cùng type + metric key, rule bị skip — tránh spam suggestion cho cùng một vấn đề.

06 — Auto-Adapt Guardrails

Giới hạn an toàn cho tự động thích nghi

Guardrail Default Mục đích
max_delta_per_cycle0.1Giới hạn mức thay đổi tham số mỗi chu kỳ
min_data_points100Cần đủ data trước khi apply — tránh overfitting
rollback_on_drop_pct20.0Auto-rollback nếu quality metric giảm >20%
locked_params[]Params không cho phép auto-thay đổi

Apply → Monitor → Rollback

Admin approve suggestion (low_retrieval_usage)
↓
Kiểm tra: min_data_points đủ? Parameter không locked? Delta ≤ 0.1?
↓
Snapshot baseline vào suggestion.parameters._baseline
↓
Apply: retrieval_threshold += max_delta_per_cycle (capped tại 0.95)
↓
OK
status → applied
Drop >20%
Auto-rollback → status → rolled_back
Chỉ threshold suggestions hỗ trợ auto-apply. Các loại tool_failure và repeated_tool là informational only — hệ thống tạo suggestion để admin đọc và quyết định thủ công. Lý do: thay đổi tool config hoặc tạo skill có tác động rộng hơn, cần human judgment.

07 — Bảo mật 4 tầng

Skill Mutations qua 4 lớp kiểm tra

Mọi thao tác tạo/sửa/xóa skill đều đi qua 4 tầng kiểm tra tuần tự:

1
Content Guard
guard.go — 25 regex rules, hard-reject on ANY violation
2
Ownership Check
Chỉ owner hoặc admin mới được sửa
3
System Skill Check
is_system=true → không thể sửa qua bất kỳ path nào
4
Filesystem Safety
symlink / path traversal / size exceeded → reject

Content Guard — 25 rules trong 6 category

Category Ví dụ bị chặn
Destructive shellrm -rf /, fork bomb, dd of=/dev/, shred
Code injectioncurl | bash, eval$(...), python -c exec()
Credential exfil/etc/passwd, .ssh/id_rsa, AWS_SECRET_ACCESS_KEY
Path traversal../../../ deep traversal
SQL injectionDROP TABLE, TRUNCATE TABLE, DROP DATABASE
Privilege escalationsudo, world-writable chmod, chown root

08 — Grants & Visibility

Kiểm soát ai dùng được skill nào

Khi skill được tạo, visibility ban đầu là private. Ngay lập tức, GrantToAgent được gọi tự động và auto-promote visibility:

Tạo skill → private (momentarily)
↓
Auto-grant cho creating agent → internal
↓
GrantToAgent / GrantToUser → internal (mở thêm)
↓
Admin promotes → public (all agents)
System Skills

Skills có is_system=true luôn public và không thể bị revoke. Grant/revoke qua HTTP API (/v1/skills/{id}/grants/agent) và WebSocket RPC.

11 — Kết luận

Ba vòng lặp tiến hóa

Self-Evolution Engine — Three Feedback Loops
Conversation-level
SOUL.md Evolution
Sau mỗi cuộc trò chuyện, agent điều chỉnh giọng nói, phong cách phản hồi. Thay đổi nhỏ tích lũy thành "nhân cách" thật sự phản ánh user preference.
Session-level
Skill Learning
Workflow phức tạp lặp lại được đóng gói thành skill tái sử dụng — với sự đồng ý của user. Agent không phải reinvent the wheel.
System-level
Metrics & Suggestions
Cron job hàng ngày phân tích 7-day window, phát hiện pattern bất thường, đề xuất cải thiện có cơ sở dữ liệu.

Ba tầng này cùng nhau tạo ra agents không chỉ thực thi task mà còn cải thiện theo thời gian — từ phong cách giao tiếp đến kho skill đến tham số hệ thống.
