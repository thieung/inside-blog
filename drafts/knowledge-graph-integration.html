Từ hội thoại thô đến mạng lưới tri thức có thể tra vứu From raw conversations to a queryable web of knowledge 从原始对话到可查询的知识网络 生の会話からクエリ可能な知識ネットワークへ

GoClaw tự động trích xuất entities, xây dựng quan hệ giữa chúng, và cho phép agent duyệt qua đồ thị tri thức nhiều bước — tất cả được hỗ trợ bởi pgvector và PostgreSQL. GoClaw automatically extracts entities, builds relationships between them, and lets agents traverse the knowledge graph across multiple hops — all powered by pgvector and PostgreSQL. GoClaw 自动提取实体、构建它们之间的关系，并允许 Agent 跨多跳遍历知识图谱 — 全部由 pgvector 和 PostgreSQL 驱动。 GoClaw はエンティティを自動抽出し、それらの関係を構築し、エージェントが複数ホップにわたる知識グラフをトラバースできるようにします — すべて pgvector と PostgreSQL により実現。

Tổng quan Overview 概述 概要

Agent nhớ quan hệ, không chỉ sự kiện Agents that remember relationships, not just facts Agent 记住关系，而不仅仅是事实 エージェントは事実だけでなく関係を記憶する

Memory thông thường lưu văn bản thô. Knowledge Graph lưu cấu trúc: người, dự án, tổ chức, sự kiện và cách chúng liên kết với nhau. Khi agent hỏi "ai làm việc với dự án GoClaw?", nó duyệt đồ thị — không cần tìm kiếm từ khóa. Regular memory stores raw text. Knowledge Graph stores structure: people, projects, organizations, events and how they connect. When an agent asks "who works on the GoClaw project?", it traverses the graph — no keyword matching needed. 普通记忆存储原始文本。知识图谱存储结构：人物、项目、组织、事件及其相互联系。当 Agent 询问"谁在参与 GoClaw 项目？"时，它遍历图谱 — 无需关键词匹配。 通常のメモリは生のテキストを保存します。ナレッジグラフは構造を保存します：人、プロジェクト、組織、イベント、そしてそれらのつながり。エージェントが「GoClaw プロジェクトに誰が関わっているか？」と尋ねると、グラフをトラバースします — キーワード検索は不要です。

Kiến trúc Architecture 架构 アーキテクチャ

Pipeline trích xuất thực thể Entity extraction pipeline 实体提取流水线 エンティティ抽出パイプライン

Mỗi lần hội thoại kết thúc, GoClaw chạy extraction pipeline bất đồng bộ. LLM phân tích ngữ cảnh và tạo JSON có cấu trúc, sau đó được ingest vào graph store với upsert semantics. Each time a conversation ends, GoClaw runs the extraction pipeline asynchronously. The LLM analyzes the context and produces structured JSON, which is then ingested into the graph store with upsert semantics. 每次对话结束时，GoClaw 异步运行提取流水线。LLM 分析上下文并生成结构化 JSON，然后通过 upsert 语义摄取到图存储中。 会話が終了するたびに、GoClaw は非同期で抽出パイプラインを実行します。LLM がコンテキストを分析して構造化 JSON を生成し、それが upsert セマンティクスでグラフストアに取り込まれます。

Traversal Traversal 遍历 トラバーサル

Duyệt đồ thị đa bậc Multi-hop graph traversal 多跳图遍历 マルチホップグラフトラバーサル

Recursive CTE trong PostgreSQL cho phép duyệt đồ thị sâu tới 3 bậc với timeout 5 giây tích hợp. Kết quả được trả về có kèm path và relation type ở mỗi bước. A recursive CTE in PostgreSQL enables graph traversal up to 3 hops deep with a built-in 5-second timeout. Results include path and relation type at each step. PostgreSQL 中的递归 CTE 支持最多 3 跳的图遍历，内置 5 秒超时。结果包含每步的路径和关系类型。 PostgreSQL のリカーシブ CTE により、最大 3 ホップのグラフトラバーサルが可能です（組み込みの 5 秒タイムアウト付き）。結果には各ステップのパスとリレーションタイプが含まれます。

Go structs cốt lõi Core Go structs 核心 Go 结构体 コア Go 構造体

store/knowledge_graph_store.go Go 
// Entity represents a node in the knowledge graph.
type Entity struct {
    ID          string            `json:"id"`
    AgentID     string            `json:"agent_id"`
    UserID      string            `json:"user_id,omitempty"`
    Name        string            `json:"name"`
    EntityType  string            `json:"entity_type"`
    Description string            `json:"description,omitempty"`
    Properties  map[string]string `json:"properties,omitempty"`
    Confidence  float64           `json:"confidence"`
}

// Relation represents a directed edge between two entities.
type Relation struct {
    ID             string  `json:"id"`
    SourceEntityID string  `json:"source_entity_id"`
    RelationType   string  `json:"relation_type"`
    TargetEntityID string  `json:"target_entity_id"`
    Confidence     float64 `json:"confidence"`
}

// TraversalResult is a connected entity with full path info.
type TraversalResult struct {
    Entity Entity   `json:"entity"`
    Depth  int      `json:"depth"`
    Path   []string `json:"path"`
    Via    string   `json:"via"`
}

Phụ thuộc dự án Project dependencies 项目依赖关系 プロジェクト依存関係

Knowledge graph tự động xây dựng cây phụ thuộc giữa các module, service, và team — agent có thể phân tích impact của một thay đổi. The graph auto-builds dependency trees across modules, services, and teams — the agent can then assess the impact of any change. 图谱自动构建跨模块、服务和团队的依赖树 — Agent 可以评估任何更改的影响。 グラフはモジュール、サービス、チームにまたがる依存関係ツリーを自動構築します — エージェントはあらゆる変更の影響を評価できます。

Lịch sử quyết định Decision history 决策历史 意思決定履歴

Mỗi quyết định kỹ thuật được lưu như một event entity, liên kết với người quyết định, dự án bị ảnh hưởng và thời điểm — truy vết về sau dễ dàng. Each technical decision is stored as an event entity linked to the decision-maker, the affected project, and the timestamp — easy to trace back later. 每个技术决策都作为事件实体存储，与决策者、受影响的项目和时间戳关联 — 后续追踪很容易。 各技術的決定は、意思決定者、影響を受けるプロジェクト、タイムスタンプにリンクされたイベントエンティティとして保存されます — 後から簡単に追跡できます。

Onboarding tự động Automated onboarding 自动化入职 自動オンボーディング
