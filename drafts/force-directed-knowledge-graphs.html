Vấn đề The Problem 问题 課題

Tại sao Force-Directed?

Why Force-Directed?

为何使用力导向布局？

なぜ力指向レイアウト？

Đồ thị tri thức (Knowledge Graph) trong GoClaw chứa các thực thể — người, tổ chức, dự án, khái niệm — liên kết với nhau qua nhiều mối quan hệ. Grid layout cứng nhắc và tree layout theo cấp bậc đều thất bại khi dữ liệu có nhiều chu trình và kết nối đan xen. Force-directed layout giải quyết vấn đề này bằng cách mô phỏng lực vật lý để tự tổ chức.

The Knowledge Graph in GoClaw holds entities — people, organizations, projects, concepts — connected through rich relationships. Rigid grid and hierarchical tree layouts both fail when data has cycles and complex cross-connections. Force-directed layout solves this by simulating physical forces for self-organization.

GoClaw 的知识图谱包含实体——人物、组织、项目、概念——通过丰富的关系相互连接。当数据存在环路和复杂交叉连接时，固定网格和层次树状布局都会失效。力导向布局通过模拟物理力来自我组织，解决了这个问题。

GoClaw の知識グラフにはエンティティ（人物、組織、プロジェクト、概念）が豊富な関係で繋がっています。データにサイクルや複雑な交差接続がある場合、固定グリッドや階層ツリーレイアウトはどちらも失敗します。力指向レイアウトは物理力をシミュレートして自己組織化することでこの問題を解決します。

TypeScript — d3-force simulation 
const simulation = forceSimulation(simNodes)
  .force("link",
    forceLink(simLinks)
      .id(d => d.id)
      .distance(220)      // Target link length in px
      .strength(0.4)     // Link rigidity
  )
  .force("charge",
    forceManyBody()
      .strength(d => -300 * (d.mass ?? 1))  // Repulsion × mass
  )
  .force("center", forceCenter(w/2, h/2))
  .force("collide",
    forceCollide()
      .radius(d => 55 + (d.mass ?? 1) * 5)
      .strength(0.8)
  )
  .stop();  // Run fixed iterations instead of continuous loop
Phân cấp Hierarchy 层级 階層

Mass Hierarchy — Orbital Layout

Mass Hierarchy — Orbital Layout

质量层级——轨道布局

質量階層——軌道レイアウト

Mỗi loại thực thể được gán một khối lượng (mass). Mass lớn hơn đồng nghĩa lực đẩy mạnh hơn — node nặng hơn sẽ ở trung tâm, còn các task nhỏ xoay quanh vòng ngoài như hành tinh.

Each entity type has an assigned mass. Greater mass means stronger repulsion — heavier nodes stay central while lightweight tasks orbit the periphery like planets.

每种实体类型都有指定的质量。质量越大，斥力越强——较重的节点保持在中心，而轻量级任务像行星一样在外围轨道运行。

各エンティティタイプには質量が割り当てられています。質量が大きいほど斥力が強く——重いノードが中央に留まり、軽量なタスクが惑星のように外周を周回します。
