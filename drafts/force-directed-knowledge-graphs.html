CUỘN XUỐNG

BỐI CẢNH

Knowledge Graph ra mắt từ v2.15.0

v2.15.0 giới thiệu Knowledge Graph — hệ thống trích xuất thực thể và quan hệ từ bộ nhớ agent, lưu trữ dưới dạng đồ thị entity-relation trong PostgreSQL. Agent có thể nhớ người dùng, tổ chức, dự án và sự kiện theo ngữ cảnh qua nhiều cuộc hội thoại.

v2.47.0 không thêm tính năng mới cho Knowledge Graph. Đây là bản phát hành tập trung hoàn toàn vào cách hiển thị: bộ render lực hướng, phân cấp khối lượng thực thể, tương tác chọn cạnh và 7 tối ưu hiệu năng. Kết quả: graph view đạt chuẩn production.

TỐI ƯU HIỆU NĂNG

7 tối ưu hóa từ v2.47.0 đến v2.47.2

1. O(1) Entity Lookup via Map

TRƯỚC

entities.find(e => e.id === id)

O(n) mỗi lần selection

SAU

useMemo(() => new Map(entities.map(e => [e.id, e])), [entities])

O(1) lookup, memo tái dùng

2. Pre-computed Edge Style Constants

TRƯỚC

Tạo edge style objects mới ở mỗi render

GC pressure khi pan/zoom

SAU

Định nghĩa EDGE_STYLE_DEFAULT và EDGE_STYLE_FADED một lần

Zero allocation khi render

3. EntityNode Memoization

TRƯỚC

Re-render tất cả nodes khi pan/zoom

SAU

memo(EntityNode)

Chỉ re-render khi data thực thể thay đổi

4. Force Simulation Tick Cap

TRƯỚC

Số tick không giới hạn

O(n²) cho graph lớn

SAU

Math.ceil(Math.log(n)) tick

O(log n) convergence

5–7. Ba tối ưu bổ sung

# Vấn đề Giải pháp
5 Double fitView — componentDidMount + manual call Single call trong useEffect
6 Fetch 200 entities, render 50 — lãng phí API Fetch 50 khớp với GRAPH_LIMIT=50
7 Theme change O(n*m) node lookup Pre-compute entityMap — O(n) total

Tất cả 7 tối ưu này tập trung vào rendering path — giảm allocation, giảm re-render không cần thiết, và đồng bộ fetch limit với render limit. Kết quả: graph 50 node render mượt mà, không jank khi pan/zoom.

Multi-tenancy — Scope Filtering

Mọi truy vấn Knowledge Graph đều được lọc theo scope. Helper scopeClause(ctx, paramIdx) tự động thêm điều kiện tenant dựa trên context.

-- Default scope (single-tenant / per user) WHERE agent_id = $1 AND user_id = $2 AND tenant_id = $3 -- Admin scope (cross-user queries) WHERE agent_id = $1 AND tenant_id = $2

ỨNG DỤNG

Graph view giải quyết gì

Khám phá quan hệ ẩn

Nhìn thấy ai liên kết với ai, dự án nào liên quan đến tổ chức nào — trực tiếp qua đồ thị.

Kiểm tra bộ nhớ agent

Xem agent đang nhớ những thực thể nào, mối quan hệ nào đang được theo dõi và điều chỉnh khi cần.

Phân tích độ sâu đồ thị

Traverse tối đa 3 hops từ bất kỳ entity nào bằng recursive CTE với color coding theo độ sâu.

Quản lý multi-tenant

Mỗi tenant được tách biệt hoàn toàn qua scope filter — admin có thể xem cross-user, người dùng chỉ thấy dữ liệu của mình.
